# 竞店商品快照模块接入 SaaS 项目对齐清单

## 1. 当前结论

第一阶段采用“嵌入 Python 执行层”的接入方式，不单独部署 HTTP 服务。

当前收口路径：

```text
Vue 竞店页面
  -> Java /api/monitor/*
  -> SQLite monitor_job
  -> backend/python/monitor_worker.py
  -> TemuMonitorAdapter
  -> embedded competitor_snapshot provider
  -> boards/ctf-website 页面卡片证据
  -> monitor_snapshot / monitor_product_snapshot / monitor_signal
```

第二阶段再考虑把同一套 `competitor_snapshot` core 包装成独立 HTTP 服务。现阶段不把 `D:/reverselab/open-reverselab` 作为 SaaS 运行时依赖。

| `SOURCE_UNAVAILABLE` | `failed` | `MONITOR_SOURCE_UNAVAILABLE` |

| `AUTH_REQUIRED` | `failed` | `MONITOR_AUTH_REQUIRED` |
| `RISK_BLOCKED` | `failed` | `MONITOR_RISK_BLOCKED` |

## 17. 下一步执行顺序

详细命令、通过标准和失败定位见 [02-生产联调运行手册.md](02-生产联调运行手册.md)。

1. 在本机确认 `scripts` 目录依赖已安装：`cd scripts && npm install`。
2. 在本机设置 `CROSSHUB_SSH_HOST`、`CROSSHUB_SSH_PASSWORD`，并设置 `CROSSHUB_MONITOR_API_TOKEN` 或 `CROSSHUB_MONITOR_LOGIN_ACCOUNT` / `CROSSHUB_MONITOR_LOGIN_PASSWORD`。
3. 运行 `node scripts/setup-remote-monitor-smoke-env.js`，写入服务器私有 `/data/crosshub/.monitor-smoke.env`。
4. 设置 `CROSSHUB_REMOTE_MONITOR_SMOKE_ENV_READY=true`。
5. 运行 `node scripts/deploy-preflight.js`，确认部署前置检查通过。
6. 运行生产部署脚本：`node scripts/deploy-server.js`。
7. 部署脚本会自动确认 `crosshub-python-worker` 容器运行中，并执行 worker 容器内 synthetic smoke。
8. 如果服务器存在 `/data/crosshub/.monitor-smoke.env`，部署脚本会通过 `node:20-alpine` Docker 容器自动执行远端 API smoke。
9. 在服务器确认 `/data/crosshub/data`、`/data/crosshub/evidence`、`/data/crosshub/reports` 与 compose 挂载一致。
10. 做一次页面联调验证：打开竞店页面，确认 synthetic smoke target 展示快照与 recent launch / sales outlier 信号。
