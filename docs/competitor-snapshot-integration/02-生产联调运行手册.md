# 竞店商品快照模块生产联调运行手册

## 1. 目的

本手册用于把竞店商品快照模块从“本机已对齐”推进到“生产已验证”。

当前生产形态固定为：

```text
Vue 竞店页面
  -> Java /api/monitor/*
  -> SQLite /data/crosshub/data/crosshub.db
  -> crosshub-python-worker
  -> /data/crosshub/evidence/raw_products.json
  -> monitor_snapshot / monitor_product_snapshot / monitor_signal
```

第一阶段不部署独立 snapshot HTTP 服务，不在 Java 容器内安装 Python，也不恢复 Playwright 竞店兜底。

## 2. 前置条件

- [ ] 本机已安装 `scripts` 目录依赖。
- [ ] 本机可通过 SSH 连接生产服务器。
- [ ] 服务器已安装 Docker，并可拉取 `node:20-alpine` 镜像。
- [ ] 服务器可保存私有 `/data/crosshub/.monitor-smoke.env`。
- [ ] 本次联调只使用 synthetic target、synthetic 商品和 `/api/monitor/*` 链路。

本机依赖安装命令：

```powershell
cd scripts
npm install
cd ..
```

## 3. 本机环境变量

本机只设置部署连接和一次性写入远端 smoke env 所需变量。

Token 方式：

```powershell
$env:CROSSHUB_SSH_HOST="your-server-host"
$env:CROSSHUB_SSH_USER="root"
Set-Item Env:CROSSHUB_SSH_PASSWORD '<private SSH password>'
Set-Item Env:CROSSHUB_MONITOR_API_TOKEN '<private API token>'
```

账号密码方式：

```powershell
$env:CROSSHUB_SSH_HOST="your-server-host"
$env:CROSSHUB_SSH_USER="root"
Set-Item Env:CROSSHUB_SSH_PASSWORD '<private SSH password>'
$env:CROSSHUB_MONITOR_LOGIN_ACCOUNT="your-login-account"
Set-Item Env:CROSSHUB_MONITOR_LOGIN_PASSWORD '<private login password>'
$env:CROSSHUB_MONITOR_PORTAL_ROLE="boss"
```

约束：

- 不把真实值写入仓库文件。
- 不把真实值写入 `deploy/monitor-smoke.env.example`。
- 不提交 `.monitor-smoke.env`。

## 4. 写入服务器私有 smoke env

执行：

```powershell
node scripts\setup-remote-monitor-smoke-env.js
```

预期结果：

- 服务器生成 `/data/crosshub/.monitor-smoke.env`。
- 终端不打印 token、账号密码明文。
- 文件由服务器私有维护，不进入 Git。

写入后设置准备完成标记：

```powershell
$env:CROSSHUB_REMOTE_MONITOR_SMOKE_ENV_READY="true"
```

## 5. 部署前置检查

执行：

```powershell
node scripts\deploy-preflight.js --help
node scripts\deploy-preflight.js
```

通过时应看到：

```text
deploy_preflight_ok
```

自动化平台可使用 JSON 输出：

```powershell
node scripts\deploy-preflight.js --json
```

JSON 会包含：

- `ok`
- `missing`
- `checks`
- `next_actions`

如果失败，按输出补齐：

- `CROSSHUB_SSH_HOST`
- `CROSSHUB_SSH_PASSWORD`
- `CROSSHUB_REMOTE_MONITOR_SMOKE_ENV_READY=true`
- `scripts dependency ssh2`
- `.monitor-smoke.env gitignore rule`
- `safe deploy/monitor-smoke.env.example`

失败输出会附带 `next_action=` 提示；JSON 输出会包含 `next_actions` 数组。提示只包含命令和环境变量名，不打印真实 SSH 密码、API token 或登录密码。

## 6. 执行生产部署

执行：

```powershell
node scripts\deploy-server.js
```

部署脚本会自动执行以下动作：

- 构建 Java JAR。
- 构建 Vue dist。
- 上传 `backend/python` 到服务器 `python-src`。
- 构建 `crosshub-python-worker:latest`。
- 启动 `crosshub-java`、`crosshub-express`、`crosshub-python-worker`。
- 创建 `/data/crosshub/data`、`/data/crosshub/evidence`、`/data/crosshub/reports`。
- 验证 `crosshub-python-worker` 容器正在运行。
- 在 worker 容器内执行 `smoke_monitor_snapshot.py`。
- 如果存在 `/data/crosshub/.monitor-smoke.env`，通过 `node:20-alpine` Docker 容器执行远端 API smoke。

## 7. 通过标准

生产部署完成后，需要同时满足：

- [ ] `crosshub-python-worker` 容器状态为 running。
- [ ] worker 容器内 synthetic smoke 通过。
- [ ] 远端 API smoke 返回 `status=ok`。
- [ ] 远端 API smoke 返回 `product_count > 0`。
- [ ] 远端 API smoke 返回 `recent_launch_count > 0`。
- [ ] 远端 API smoke 返回 `sales_outlier_count > 0`。
- [ ] `/data/crosshub/data/crosshub.db` 被 Java 和 worker 共用。
- [ ] `/data/crosshub/evidence` 可被 API smoke 写入、被 worker 读取。
- [ ] `/data/crosshub/reports` 可被 worker 写入。
- [ ] 前端竞店页面能通过 `/api/monitor/*` 查询到最新快照和信号。

## 8. 失败定位

### 8.1 preflight 失败

先按 `deploy_preflight_failed=` 后面的字段补齐配置。

常见原因：

- 未设置 SSH 环境变量。
- 未安装 `scripts` 目录依赖。
- 未运行 `setup-remote-monitor-smoke-env.js`。
- 未设置 `CROSSHUB_REMOTE_MONITOR_SMOKE_ENV_READY=true`。

### 8.2 worker 容器未运行

在服务器检查：

```bash
docker ps --filter name=crosshub-python-worker
docker logs crosshub-python-worker --tail 200
```

重点看：

- `CROSSHUB_DB_PATH=/data/crosshub.db`
- `CROSSHUB_MONITOR_EVIDENCE_DIR=/evidence`
- SQLite 文件是否可读写
- `/evidence` 是否挂载成功

### 8.3 worker synthetic smoke 失败

在服务器复跑：

```bash
docker exec crosshub-python-worker python smoke_monitor_snapshot.py --work-dir /tmp/monitor-smoke
```

重点看：

- synthetic `raw_products.json` 是否生成。
- `monitor_snapshot` 是否写入。
- `monitor_signal` 是否生成 recent launch / sales outlier。

### 8.4 远端 API smoke 失败

在服务器复跑同类命令：

```bash
docker run --rm --network host \
  -v /data/crosshub/scripts:/scripts:ro \
  -v /data/crosshub/evidence:/data/crosshub/evidence \
  node:20-alpine \
  node /scripts/monitor-api-smoke.js \
    --base-url http://127.0.0.1:18080 \
    --evidence-root /data/crosshub/evidence \
    --db-path /data/crosshub/data/crosshub.db \
    --no-local-worker \
    --timeout-ms 60000
```

重点看：

- Java `:18080` 是否可访问。
- `/data/crosshub/.monitor-smoke.env` 是否存在并已在部署脚本中加载。
- synthetic target 是否创建成功。
- evidence 是否写入到 `/data/crosshub/evidence`。
- worker 是否消费 pending job。

## 9. 边界

- 不抓真实 Temu URL。
- 不保存真实 Cookie、token、请求响应日志。
- 不把生产 trace 写入仓库。
- 不把 Open ReverseLab 工作区作为 SaaS 运行时依赖。
- 不引入 Playwright 兜底。
