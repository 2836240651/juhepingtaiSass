# Temu 一键刷新爬取 — 技术设计

## 架构

```
Vue TemuModuleView
  → POST /api/temu/crawl
  → TemuCrawlService (afterCommit + crawlExecutor)
  → ProcessBuilder: py crawl.py --tenant-id N --json
  → SQLite ingest
  ← GET /api/temu/crawl/{id} 轮询
  ← GET /api/temu/operational 重载
```

## 核心组件

| 组件 | 路径 |
|------|------|
| 任务表迁移 | `V3TemuCrawlJobMigration.java` |
| 销售唯一约束 | `V4TemuSaleTenantUniqueMigration.java` |
| 爬虫配置 | `CrawlerProperties` + `application.yml` |
| 业务服务 | `TemuCrawlService.java` |
| 权限 | `TemuCrawlAuthService.java` |
| API | `TemuCrawlController.java` |
| 异步线程池 | `AsyncConfig.java` |
| Python 入口 | `crawl.py --json` |
| 前端 API | `temuApi.js` |
| 前端 UI | `TemuModuleView.vue` |

## 数据模型 `temu_crawl_job`

- `status`: pending | running | success | failed
- `mode`: live | seed
- 索引：`(tenant_id, status)`

## 关键实现细节

1. **事务后调度**：`triggerCrawl` 在 `@Transactional` 内保存 job，通过 `TransactionSynchronization.afterCommit` 再提交异步任务，避免 worker 读不到未提交记录。
2. **Python 路径**：Java 从 `backend/java` 启动时 `script-dir` 为 `../python`。
3. **多租户唯一键**：`temu_sale` 唯一约束为 `(tenant_id, report_time, shop_id, ext_code)`。

## 配置

```yaml
crosshub:
  crawler:
    python-executable: py          # 生产 python3
    script-dir: ../python
    timeout-seconds: 300
    allow-seed: true               # 生产 false
```
