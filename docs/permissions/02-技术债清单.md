## P0 — 必须先还

### TD-PERM-01：Demo 菜单双源，与 `sys_menu` 易漂移

| 项 | 内容 |
|----|------|
| 问题 | `menuAuth.js` 的 `fallbackSidebarMenus` 与 `TenantSchemaMigration.seedMenus` 两套维护；改菜单需改两处 |
| 影响 | 联调/Dev 行为不一致；新增路由 `menuCode` 与侧栏条目可能脱节 |
| 关键文件 | `dev/vue-site/src/utils/menuAuth.js`、`backend/java/.../config/migration/TenantSchemaMigration.java` |
| 修复方向 | M3：导出 `sys_menu` → `constants/menus.js` 或 Demo 登录写入菜单快照；`resolveSidebarMenus` 单源 |
| 阻塞上线 | 是（多环境演示易误导） |

### TD-PERM-02：`shop_scope` ID 空间混用

| 项 | 内容 |
|----|------|
| 问题 | `user_shop_scope.shop_id` 存 Temu `temu_shop.shop_id`；运营绑定与 Demo 用 `platform_account.id` |
| 影响 | Boss 在运营绑定页分配的店铺与 Temu 后端 scope 可能对不上；员工 Temu 页与绑定页 ID 不一致 |
| 关键文件 | `MemberScopeServiceImpl.java`、`DataScopeServiceImpl.java`、`dev/vue-site/src/utils/scope.js` |
| 修复方向 | M4：统一主键模型（映射表或 scope 存 platform_account.id + 平台字段） |
| 阻塞上线 | 是（后端模式 Temu 多店铺场景） |

## P1 — 功能与一致性缺口

### TD-PERM-03：无 `tenant_feature` 管理 API/UI

| 项 | 内容 |
|----|------|
| 问题 | 仅有注册时 `seedTenantFeatures` 写入；无 Controller；Boss 无法裁切功能 |
| 影响 | 注册即全开所有平台菜单；无法做套餐/试用裁剪 |
| 关键文件 | `entity/TenantFeature.java`、`MenuServiceImpl.java`（读 feature） |
| 修复方向 | M2：`GET/PUT /api/tenant/features` + Boss 设置子页 |
| 阻塞上线 | 否（演示可接受） |

### TD-PERM-04：非 Temu 无后端数据权限

| 项 | 内容 |
|----|------|
| 问题 | Amazon/Walmart 等数据在 `*Local.js`；`scope.js` 仅前端过滤 |
| 影响 | 员工改 URL 或未来接 Java API 时无服务端校验 |
| 关键文件 | 各 `*ModuleView.vue`、`scope.js`、各 `*Local.js` |
| 修复方向 | Phase 3 平台扩展时每个真平台补 Service scope |
| 阻塞上线 | 否（当前为 Demo） |

### TD-PERM-05：`assignable-menus` 仅一项

| 项 | 内容 |
|----|------|
| 问题 | `assignableEmployeeMenuCodes()` 仅返回 `employee.warehouse` |
| 影响 | Boss 对员工「菜单授权」模型理解成本高；平台菜单只能靠 platforms 隐式推导 |
| 关键文件 | `MemberScopeServiceImpl.java`（约 267–270 行）、`EmployeeBindingView.vue` |
| 修复方向 | 文档化策略；或扩展 assignable 含 `employee.ai` 等 |
| 阻塞上线 | 否 |

### TD-PERM-06：Demo 路由守卫弱于后端

| 项 | 内容 |
|----|------|
| 问题 | Demo 下 `canAccessRoute` 对 Boss/Warehouse 返回 true；员工靠 path 白名单 |
| 影响 | Demo 员工可能通过直链访问未授权 `menuCode` 路由 |
| 关键文件 | `menuAuth.js`、`router/index.js` |
| 修复方向 | M3：Demo 也按 `menuCode` + 计算 `menuPaths` 判定 |
| 阻塞上线 | 否（演示环境） |

## P2 — 运维与扩展

### TD-PERM-07：`sys_menu` 更新不同步 `tenant_feature`

| 项 | 内容 |
|----|------|
| 问题 | 迁移新增 menu code 后，已有租户 `tenant_feature` 无对应行 |
| 影响 | 新菜单老租户不可见；需手工 SQL |
| 关键文件 | `TenantSchemaMigration.java` |
| 修复方向 | 迁移后 `INSERT OR IGNORE` backfill 全租户；或 feature 版本号 |
| 阻塞上线 | 否 |

### TD-PERM-08：无权限相关自动化测试

| 项 | 内容 |
|----|------|
| 问题 | Java/Vue 均无菜单与 scope 单测或集成测 |
| 影响 | 重构 `MenuServiceImpl` / `menuAuth.js` 易回归 |
| 关键文件 | — |
| 修复方向 | Phase 4：`MenuServiceImplTest`、`menuAuth` Vitest smoke |
| 阻塞上线 | 否 |
