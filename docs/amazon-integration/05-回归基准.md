## 2. 不可破坏项（Hard Regression）

| HR-02 | `GET /api/platform-accounts?platform=temu` | 200 |
| HR-03 | `GET /api/temu/shops` | 200（或 401 未登录） |

## 4. 数据基线快照（联调成功后记录）

首次 P1 通过后，在 `crosshub.db` 记录以下基线供后续 diff：

| 表 | 记录条件 | 示例 |
|----|----------|------|
| `platform_account` | platform=amazon 且 external_shop_id 非空 | 1 行，external=16505337258263 |
| `amazon_sync_job` | 最近 success | scope=account_health |
| `amazon_account_metric` | 每店 ≥1 行 | status ∈ normal/warning/critical |

**禁止**回归基准中出现：

- `store_id` / `shop_id` 为 `demo_amazon_*`
- `temu_sale` 或 Temu 相关表被 Amazon 迁移误删
- localStorage `crosshub_amazon_*` 在后端模式被写入 Demo
