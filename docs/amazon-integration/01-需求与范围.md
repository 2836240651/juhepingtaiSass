# Amazon 联调 — 需求与范围

## 1. 业务目标

在 **后端模式**（`VITE_USE_TEMU_BACKEND=true` + 已登录 Java）下，CrossHub Amazon 模块从 Demo 切换为 **真实数据**：

1. **紫鸟店铺绑定**：将紫鸟 `browserId` / `browserOauth` 与 `platform_accounts` 关联
2. **数据同步**：通过紫鸟隔离环境拉取 Seller Central 报表/账户状况，入库后供 Vue 展示
3. **刷新闭环**：页面「刷新」→ 异步任务 → 轮询 → 重载（对齐 Temu/AliExpress 模式）
4. **运营总览**：Amazon 板块展示真实待办计数，不再注入 `amazonDailyLocal` 种子

## 2. 用户故事（P0～P2）

| ID | 角色 | 故事 | 验收 |
|----|------|------|------|
| US-01 | Boss | 从紫鸟导入店铺并绑定到 CrossHub | 绑定表含 `ziniao_browser_id` |
| US-02 | Boss | 查看 Agent 节点在线状态 | 离线时提示，不展示 Demo |
| US-03 | 运营 | 打开 Amazon 日报，看到真实账户状况 | 非 `demo_amazon_*` storeId |

| US-05 | Boss | 产品 TOP / 出库单来自同步结果 | Boss Tab 有数据或明确空态 |
| US-06 | 员工 | 仅能看到 scope 内 Amazon 店铺数据 | 越权 403/空 |

## 3. 范围（In Scope）

### P0 — 绑定与基础设施

- `integration_agent` 注册/心跳
- 紫鸟 `getBrowserList` → CrossHub 可选店铺列表
- `platform_accounts` 扩展外部 ID（紫鸟 browserId）
- `platformOperationalMode.js`：Agent 在线 + 已绑定 → 非 Demo
- Vite 代理 `/api/amazon` → `:18080`

### P1 — 日报 MVP

- `POST /api/amazon/sync`（scope: `account_health` | `daily`）
- Agent：`startBrowser` → Playwright CDP → 下载/解析 Account Health
- `GET /api/amazon/daily` 七类资源（先实现 `account_metrics`，其余可空数组）
- 前端 `amazonApi.js` 替换 `amazonDailyLocal.js` 读路径

### P2 — Boss 洞察 + 报表扩展

- scope: `reports`（Business Report CSV）、`messages`（买家消息列表页）
- `GET /api/amazon/insights`（products + outbound）
- 运营总览 `buildAmazonSection` 接真实 API

### P3 — 写操作与 SP-API（规划）

- PATCH messages/reviews/cases（状态回写，可先仅 CrossHub 侧标记）
- SP-API OAuth 在紫鸟窗口完成（refresh_token 存 Java）

## 4. 非目标（Out of Scope）

- 替代紫鸟做 IP/指纹隔离（继续依赖紫鸟设备）
- 在 **云端服务器** 裸 Playwright 登录多个 Amazon 账号
- 一期不做虎步 RPA API 对接（文档预留接口位）
- 不改变 `VITE_USE_TEMU_BACKEND=false` 时现有 Demo 行为
- 不修改紫鸟 Boss 账号密码策略（自动化子账号仅 WebDriver）

## 6. 成功标准（一期）

- [ ] 后端模式下 Amazon 页 **零** `amazonDailyLocal` / `amazonBossLocal` 种子注入
