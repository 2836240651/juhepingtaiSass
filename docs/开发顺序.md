# CrossHub 项目全局审查与开发顺序

> 本文档由全局审查计划落盘，供团队按阶段执行。项目根目录：`SaaS-HZ_WEB_Demo`。

## 一、项目全景

| 子项目 | 路径 | 职责 |
|--------|------|--------|
| 前端 | `dev/vue-site/` | 三门户 UI、Demo 数据、Java 联调开关 |
| Java API | `backend/java/` | 认证、租户、仓库、Temu 读/爬、平台账号 |
| Python | `backend/python/` | Temu Playwright 爬取入库 |
| Express | `script/api-server/` | 健康检查桩（未迁移 Demo 接口） |
| 数据 | `backend/data/crosshub.db` | 四栈共用 SQLite |

**代理规则**（`dev/vue-site/vite.config.js`）：`/api/temu`、`/api/auth`、`/api/warehouse`、`/api/tenant`、`/api/platform-accounts` → Java `:18080`；其余 `/api` → Express `:3000`。

**后端模式**：`dev/vue-site/.env` 中 `VITE_USE_TEMU_BACKEND=true` 时，Boss 登录后核心主数据走 Java + SQLite；默认 `false` 为纯前端 Demo。

| 模块 | 前端路由 | 数据来源 | Java API | 真实爬取 | 租户隔离 |

| 登录/注册 | `/login` `/register` | Java + Local 回退 | 有 | — | JWT `tid` |

| Temu 运营 | `/boss\|employee/temu` | Java + Local + 刷新爬取 | `/api/temu/*` + crawl | **有** | 四栈已对齐 |
| 仓库下单/分仓/人员 | warehouse + boss | Java + Local | `/api/warehouse/*` | — | 已对齐 |
| 任务分配/反馈 | boss + employee + warehouse tasks | **仅 Local** | 无 | — | tenantStorage |
| AliExpress / Amazon / Walmart / 1688 / DTC | 各 ModuleView | **仅 Local Demo** | 无 | 前端模拟刷新 | tenantStorage |
| 拼多多 / 抖音 / 视频号 | domestic 模块 | Local（factory 已 tenant） | 无 | 模拟 | 已对齐 |

**结论**：当前唯一「真后端 + 真爬取」闭环是 **Temu**；仓库、租户人事、账户绑定在 `VITE_USE_TEMU_BACKEND=true` 时可走 Java；其余平台均为 **演示层**。

## 四、推荐开发顺序

原则：**先稳住基线 → 补齐四栈断层 → 扩展真后端能力 → 工程化与上线**。

### Phase 0：稳定基线 — 已完成

| 序号 | 任务 | 完成标准 |
|------|------|----------|
| 0.1 | 7 个 `*Local.js` 接入 `tenantStorage.js` | `npm run build` 通过；租户 A/B 数据不串 |
| 0.2 | 禁止 PowerShell 批量改 JS；单文件 + build 门禁 | 团队约定 |
| 0.3 | 同步 README / AGENTS / backend README | session、crawl、platform-accounts |
| 0.4 | 固化 `start-local.ps1` + `VITE_USE_TEMU_BACKEND` 说明 | `.env.example`、AGENTS.md |
| 0.5 | Temu seed 爬取回归 | 见 `docs/temu-crawl/04-测试用例清单.md` |

### Phase 1：四栈对齐补全 — 核心已完成

| 序号 | 任务 | 说明 |
|------|------|------|
| 1.1 | 账户绑定接 Java | `platformAccountsApi.js` + facade 切换 |
| 1.2 | 厘清 Express 边界 | Express 仅 `/api/health`；店铺绑定归 Java |
| 1.3 | Temu live 爬取运维包 | AGENTS.md、deploy 补充 Python/Profile |
| 1.4 | Java 小清理 | 统一 `ApiResult`；整理 Controller 格式 |

**完成标准**：后端模式下账户绑定 CRUD 与 SQLite 一致；Express 与 Java 无重复写同一业务。

### Phase 2：Temu 产品化 — 进行中

| 序号 | 任务 | 说明 |
|------|------|------|
| 2.1 | 爬取 UX | 409 并发提示、失败 `error_message`、login 指引 |
| 2.2 | 爬取任务历史 | GET 最近 N 次 `temu_crawl_job` + 简单 UI |
| 2.3 | 生产 Docker | Java 挂载 Python、Profile、DB |
| 2.4 | 全量测试 | 执行 `docs/temu-crawl` 用例清单 |

### Phase 3：平台扩展策略（按业务选型）

在 Temu 稳定前，**其它平台维持 Demo**。

若需第二个真平台，推荐顺序：

1. **AliExpress** — 可复用 Temu crawl 任务模式
2. **Walmart** — 订单 + Listing 双 Tab
3. **Amazon** — 紫鸟 WebDriver + Agent（见 [`docs/amazon-integration/`](../amazon-integration/README.md)，PoC 已通）
4. **1688 / DTC** — 靠后

每个真平台需：**爬虫或 API → Java 读 API → Vue backend 开关 → 租户隔离**。

**任务分配（可选）**：设计 `assigned_task` 表 + `/api/tasks`，替换 `assignedTasksLocal`。

### Phase 4：工程化与上线硬化（持续）

| 序号 | 任务 |
