> 项目根目录：`SaaS-HZ_WEB_Demo`  
> 关联：`01-未联调清单.md`、`03-PRD实施包.md`、`04-测试用例.md`  
> 更新基准：2026-07-06（P0～P3 已实施；P4 策略 A 已门禁 Demo）

## 背景

默认模式为 **后端模式**（`VITE_USE_TEMU_BACKEND` 未显式设为 `false`）。Vite 将 `/api/auth`、`/api/tenant`、`/api/warehouse`、`/api/platform-accounts`、`/api/temu`、`/api/tasks`、`/api/ops-feedback` 代理至 Java `:18080`。

**Temu 运营读数、账户绑定、出库单、任务/反馈、平台推仓** 已接 Java；**其余 8 个平台运营数据** 在后端模式下已门禁 Demo（仅展示绑定店铺 +「接口开发中」提示），**Java 运营 API 尚未建设**。

## 五、新建 Java API 统一规范

| 项 | 约定 |
|----|------|
| 路径前缀 | `/api/{platform}` 或 `/api/domestic/{platform}` |
| 租户隔离 | 从 JWT `tenant_id` 过滤 |
| 店铺 scope | `platform_account.id` + 可选 `external_shop_id` |
| 查询参数 | `store_id`、`refresh=true`（触发同步） |
| Record DTO | snake_case（`store_id`, `synced_at`） |
| Map body（出库单类） | camelCase 保持原样 |
| 响应 | 前端 `map*` 双读 camel/snake |
| Vite 代理 | `vite.config.js` 增加 `/api/aliexpress` 等 → `:18080` |
| 前端双模式 | `hasBackendSession(auth)` 走 `*Api.js`，否则 `*Local.js` |

### 建议通用表结构

```
platform_order               — 各平台订单快照
platform_issue               — 违规/Listing/活动预警等
platform_operational_snapshot — 商品/流量等聚合（可选）
```

公共列建议：`tenant_id`, `platform`, `store_id`（绑定表 UUID）, `external_ref`, `payload_json`, `synced_at`。

## 六、推荐实施顺序

```
1. 国内三平台（PDD / 抖音 / 视频号）— 结构相同，可一次做 DomesticController
2. Walmart + AliExpress — 跨境订单 + 问题，模式接近
3. 1688 — 采购单链路，推仓已通
4. DTC — 商品 / 流量 / 营销
5. Amazon — 子资源最多，拆 daily / insights 两阶段
6. Temu 竞品 + 备货 — 独立 Temu 子域
```
