# 通用竞店监控框架测试用例清单

## 1. API 用例

| ID | 场景 | 预期结果 |
|---|---|---|
| API-01 | 创建监控目标 | 200，返回目标主键与基础信息 |
| API-02 | 更新监控目标 | 200，字段更新生效 |
| API-03 | 删除监控目标 | 200，目标不可再查询 |
| API-04 | 列出监控目标 | 仅返回当前租户目标 |
| API-05 | 查询目标详情 | 返回基础信息与最新快照指针 |
| API-06 | 配置 interval schedule | 200，返回 `next_run_at` |
| API-07 | 配置 cron schedule | 200，返回 `next_run_at` |
| API-08 | 手动 trigger 成功 | 202，返回 `job_id`，状态为 `pending` |
| API-09 | 活跃任务冲突 trigger | 409，错误码为 `MONITOR_JOB_IN_PROGRESS` |
| API-10 | latest 命中新鲜快照 | `has_fresh_data=true` |
| API-11 | latest 命中过期快照 | `has_fresh_data=false` 且可判断是否允许 trigger |
| API-12 | latest 从未抓取 | `has_fresh_data=false`，`latest_snapshot_id=null` |
| API-13 | 查询 job 状态 | 可返回 pending/running/success/failed |
| API-14 | 查询 history | 返回 `snapshots` 与 `jobs` |
| API-15 | 未登录访问 | 401 |
| API-16 | 跨租户读取 target | 404 或无权限 |
| API-17 | 跨租户读取 job | 404 或无权限 |

## 2. Scheduler 用例

| ID | 场景 | 预期结果 |
|---|---|---|
| SCH-01 | enabled=1 且 `next_run_at <= now` | 自动创建 `pending` job |
| SCH-02 | enabled=0 | 不创建 job |
| SCH-03 | 目标已有活跃任务 | 不重复创建 job |
| SCH-04 | interval 计划触发后 | `next_run_at` 正确顺延 |
| SCH-05 | cron 计划触发后 | `next_run_at` 正确按 cron 计算 |

## 3. Worker 用例

| ID | 场景 | 预期结果 |
|---|---|---|
| WK-01 | Worker 拉取 `pending` job | 任务被更新为 `running` |
| WK-02 | 正常抓取成功 | job=`success`，写入 `snapshot_id` |
| WK-03 | adapter 抛错 | job=`failed`，写入错误码与错误消息 |
| WK-04 | 同一 job 并发消费 | 仅一个 worker 抢占成功 |
| WK-05 | 成功后回填 target 最新快照 | `latest_snapshot_id/latest_snapshot_at` 更新 |
| WK-06 | 本次失败但历史快照存在 | latest 仍可读旧快照，最新 job 为 failed |

## 4. 数据落库用例

| ID | 场景 | 预期结果 |
|---|---|---|
| DB-01 | 创建目标 | `monitor_target` 插入成功 |
| DB-02 | 配置计划 | `monitor_schedule` upsert 成功 |
| DB-03 | 创建任务 | `monitor_job` 插入成功 |
| DB-04 | 成功抓取落库 | `monitor_snapshot` / `monitor_product_snapshot` / `monitor_signal` 均有数据 |

| DB-06 | 信号落库 | `recent_launch` / `sales_outlier` 计数正确 |
| DB-07 | 跨租户查询 | 数据互不污染 |

| ID | 场景 | 预期结果 |
|---|---|---|

## 8. 建议执行顺序

1. Java 单测与 API 测试
2. Python Worker 单测
3. 前端 build 与页面联调
4. 手工跑通端到端链路：
   - create target
   - trigger
   - worker success
   - latest hit
   - history visible
